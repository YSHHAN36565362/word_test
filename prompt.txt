'scheme (스키마) - 1. 요구사항 확인

인터넷 리소스에 접근하는 방법이나 통신 프로토콜 규칙을 가리킨다. (치트키 기호: 맨 앞 : 앞부분)



authority (기관/권한) - 1. 요구사항 확인

서버의 호스트 이름(도메인 주소)과 포트 번호 등을 포함하는 영역이다. (치트키 기호: // 바로 뒷부분)



path (경로) - 1. 요구사항 확인

서버 내에서 찾고자 하는 특정 자원이나 파일이 위치한 디렉터리 경로를 가리킨다. (치트키 기호: 글자 사이 사이의 /)



query (쿼리/질의어) - 2. 화면 설계

서버가 처리할 수 있도록 웹 페이지에 전달하는 추가적인 데이터 키-값 쌍이다. (치트키 기호: ? 뒷부분)



fragment (조각/책갈피) - 2. 화면 설계

웹 페이지 화면을 불러온 뒤, 그 문서 내부의 특정 위치로 바로 스크롤 시키기 위한 식별자이다. (치트키 기호: # 뒷부분)
