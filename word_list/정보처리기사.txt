살충제 패러독스
동일한 테스트 케스트에 의한 반복적 테스트는 새로운 버그를 찾지 못한다는 테스트의 원리

데이터 마이닝
대규모로 저장된 데이터 안에서 체계적이고 자동적으로 통계적 규칙이나 패턴을 찾아내는 기술 (광석에서 극히 소량으로 포함된 금을 추출하듯이 '수많은 데이터에서 가치 있는 정보를 찾아내는 것')

구문(Syntax)
시스템 간의 정보 전송을 위한 데이터 형식, 코딩, 신호 레벨 등을 규정 (프로토콜의 3요소 중 하나)

의미(Semantic)
시스템 간의 정보 전송을 위한 제어 정보로 조정과 에러 처리를 위한 규정 (프로토콜의 3요소 중 하나)

타이밍(Timing)
시스템 간의 정보 전송을 위한 속도 조절과 순서 관리 규정 (프로토콜의 3요소 중 하나)

프로토콜(Protocol)
복수의 컴퓨터 사이에서 데이터 통신을 원활하게 하기 위해 필요한 통신규약 (대표적으로 IP/TCP가 있음)

XML
송·수신 시스템 간 데이터 연계의 편의성을 위해서 전송되는 데이터 구조를 동일한 형태로 정의

JSON
비동기 브라우저·서버 통신(AJAX)을 위해 '속성-값 쌍', '키-값-쌍'으로 이루어진 데이터 오브젝트를 전달하기 위해 인간이 읽을 수 있는 텍스트를 사용하는 개방형 표준 포맷

우선순위(Priority)
프로세스별로 우선순위가 주어지고, 우선순위에 따라 CPU를 할당하는 비선점형 스케줄링 기법

기한부(Deadline)
작업들이 명시된 시간이나 기한 내에 완료되도록 계획하는 비선점형 스케줄링 기법

FCFS(First Come First Service)
프로세스가 대기 큐에 도착한 순서에 따라 CPU를 할당하는 비선점형 스케줄링 기법

SJF(Shortest Job First)
프로세스가 도착하는 시점에 따라 그 당시 가장 작은 서비스 기간을 갖는 프로세스가 종료까지 자원을 점유하는 비선점형 스케줄링 기법

HRN(Highest Response Ratio Next)
대기 중인 프로세스 중 현재 응답률이 가장 높은 것을 선택하는 비선점형 스케줄링 기법 [(대기 시간 + 서비스 시간) / 서비스 시간]

비선점형 스케줄링
하나의 프로세스가 끝나지 않으면 다른 프로세스는 CPU를 사용할 수 없다는 의미의 스케줄링 방식

RR(Round Robin)
시분할 시스템에서 사용되며 FCFS와 비슷하지만 제한시간이 지난 후 다음 프로세스에게 자원을 할당하는 선점형 스케줄링 기법

SRT(Shortest Remaining Time)
SJF 기법을 선점형으로 바꾼 스케줄링 기법

선점 우선순위
비선점 우선순위를 선점형으로 바꾼 스케줄링 기법

다단계 큐
프로세스의 우선순위에 따라 시스템, 대화형, 일괄처리 프로세스 등으로 나누어 준비상태 큐를 상·중·하위 단계로 배치하는 선점형 스케줄링 기법

다단계 피드백 큐
다단계 큐의 단점을 보완하여 큐마다 timeout을 설정하고, 초과 시 우선순위가 낮은 다음 단계 큐로 이동시키는 선점형 스케줄링 기법

선점형 스케줄링
하나의 프로세스가 다른 프로세스 대신에 프로세서(CPU)를 강제로 차지할 수 있다는 의미의 스케줄링 방식

원자성(Atomicity)
분해가 불가능한 작업의 최소단위라는 트랜잭션의 특성

일관성(Consistency)
트랜잭션이 실행 성공 후 항상 일관된 데이터베이스 상태를 보존해야 하는 특성

격리성/독립성(Isolation)
트랜잭션 실행 중 생성하는 연산의 중간 결과를 다른 트랜잭션이 접근 불가한 특성

영속성/지속성(Durability)
성공이 완료된 트랜잭션의 결과는 영속적으로 데이터베이스에 저장되어야 하는 특성

트랜잭션(Transaction)
데이터베이스의 상태를 변화시키는 하나의 논리적 기능을 수행하기 위한 작업의 단위 또는 한꺼번에 모두 수행되어야 할 일련의 연산

커밋(Commit)
트랜잭션을 확정하고 메모리에 영구적으로 저장하는 TCL 명령어

롤백(Rollback)
트랜잭션을 취소하고 내역을 무효화시키는 TCL 명령어

체크 포인트(Check Point)
저장 시기를 설정하여 롤백을 위한 시점을 지정하는 TCL 명령어

랜드 어택(Land Attack)
출발지(Source) IP와 목적지(Destination) IP를 같은 패킷 주소로 만들어 보냄으로써 수신자가 자기 자신에게 응답을 보내게 하여 시스템의 가용성을 침해하는 공격 기법

MD5 (Message-Digest algorithm 5)
각각의 512bit짜리 입력 메세지 블록에 대해 차례로 동작하여 128bit의 해시값을 생성하는 해시 알고리즘

SHA-1 (Secure Hash Algorithm-1)
160bit의 해시값을 생성하는 해시 알고리즘

SHA-256/384/512
SHA 알고리즘의 한 종류로써 각각 256/384/512bit의 해시값을 생성하는 해시 함수

HAS-160
국내 표준 서명 알고리즘 KCDSA를 위하여 개발된 해시 함수

내용 결합도(Content Coupling)
다른 모듈 내부에 있는 변수나 기능을 다른 모듈에서 사용하는 경우의 결합도

공통 결합도(Common Coupling)
파라미터가 아닌 모듈 밖에 선언되어 있는 전역 변수를 참조하고 갱신하는 식으로 상호작용하는 경우의 결합도

외부 결합도(External Coupling)
두 개의 모듈이 외부에서 도입된 데이터 포맷, 통신 프로토콜, 또는 디바이스 인터페이스를 공유할 경우의 결합도

제어 결합도(Control Coupling)
단순 처리할 값만 전달되는 게 아니라 어떻게 처리를 해야 한다는 제어 요소가 전달되는 경우의 결합도

스탬프 결합도(Stamp Coupling)
모듈 간의 인터페이스로 배열이나 객체, 구조 등이 전달되는 경우의 결합도

자료 결합도(Data Coupling)
모듈 간의 인터페이스로 전달되는 파라미터를 통해서만 모듈간의 상호작용이 일어나는 경우의 결합도

결합도 순서
내용 결합도(품질 낮음) - 공통 결합도 - 외부 결합도 - 제어 결합도 - 스탬프 결합도 - 자료 결합도(품질 좋음)

결합도(Coupling)
어떤 모듈이 다른 모듈에 의존하는 정도를 나타내는 지표

우연적 응집도(Coincidental Cohesion)
모듈 내부의 각 구성요소가 연관이 없을 경우의 응집도

논리적 응집도(Logical Cohesion)
유사한 성격을 갖거나 특정 형태로 분류되는 처리 요소들이 한 모듈에서 처리되는 경우의 응집도

시간적 응집도(Temporal Cohesion)
연관된 기능이라기보다는 특정 시간에 처리되어야 하는 활동들을 한 모듈에서 처리할 경우의 응집도

절차적 응집도(Procedural Cohesion)
모듈이 다수의 관련 기능을 가질 때 모듈 안의 구성요소들이 그 기능을 순차적으로 수행할 경우의 응집도

교환적/통신적 응집도(Communication Cohesion)
동일한 입력과 출력을 사용하여 다른 기능을 수행하는 활동들이 모여 있을 경우의 응집도

순차적 응집도(Sequential Cohesion)
모듈 내에서 한 활동으로부터 나온 출력값을 다른 활동이 연이어 사용할 경우의 응집도

기능적 응집도(Functional Cohesion)
모듈 내부의 모든 기능이 단일한 목적을 위해 수행되는 경우의 응집도

응집도 순서
우연적 응집도(품질 낮음) - 논리적 응집도 - 시간적 응집도 - 절차적 응집도 - 통신적 응집도 - 순차적 응집도 - 기능적 응집도(품질 좋음)

응집도(Cohesion)
모듈에 포함된 내부 요소들이 하나의 책임과 목적을 위해 연결되어 있는 연관된 정도

헤더(Header)
문서 이름, 제품 이름, 버전 번호, 릴리즈 날짜, 참고 날짜, 노트 버전 등의 릴리즈 노트 정보

개요(Overview)
제품 및 변경에 대한 간략한 전반적 개요를 담은 릴리즈 노트 항목

목적(Purpose)
새로운 기능 목록과 릴리스 노트의 목적에 대한 개요, 버그 수정 및 새로운 기능 기술

이슈 요약(Issue Summary)
버그의 간단한 설명 또는 릴리즈 추가 항목 요약

재현 항목(Reproduction)
버그 발견에 따른 재현 단계를 기술한 항목

수정·개선 내용
수정·개선 사항에 대한 간단한 설명을 기술한 항목

사용자 영향도
버전 변경에 따른 최종 사용자 기준의 기능 및 응용 프로그램상의 영향도 기술

소프트웨어 지원 영향도
버전 변경에 따른 소프트웨어의 지원 프로세스 및 영향도 기술

노트(Notes)
소프트웨어 및 하드웨어 설치 항목, 제품 문서를 포함한 업그레이드 항목 메모

면책 조항(Disclaimer)
회사 및 표준 제품과 관련된 메세지, 프리웨어 및 불법 복제 방지, 중복 등 참조에 대한 고지 사항

연락 정보(Contact Info)
사용자 지원 및 문의에 관련한 연락처 정보

릴리스 노트(Release Note)
소프트웨어 제품과 함께 배포되며 제품의 변경 및 개선 사항을 사용자에게 알리는 문서

테이블 병합
1:1관계, 1:M 관계를 통합하여 조인 횟수를 줄여 성능을 향상시키는 반정규화 기법

테이블 분할(파티셔닝)
테이블을 수직 또는 수평으로 분할하여 성능을 향상시키는 반정규화 기법

중복 테이블 추가
대량의 데이터 집계 함수 계산을 위해 별도의 통계 테이블이나 중복 테이블을 추가하는 반정규화 기법

컬럼 중복화
조인 성능 향상을 위해 의도적으로 컬럼의 중복을 허용하는 반정규화 기법

중복 관계 추가
성능 저하를 예방하기 위해 데이터를 처리하는 여러 경로에 추가적인 관계를 맺는 반정규화 방법

반정규화(De-normalization)
시스템의 성능 향상, 개발 및 운영의 편의성을 위해 정규화된 데이터 모델을 의도적으로 통합, 중복, 분리하여 정규화 원칙을 위배하는 행위

응용 계층(Application Layer)
사용자와 네트워크 간 응용서비스 연결, 데이터를 생성하는 OSI 7계층 (7계층)

표현 계층(Presentation Layer)
데이터 형식 설정, 부호교환, 암호화 및 복호화를 담당하는 OSI 7계층 (6계층)

세션 계층(Session Layer)
송수신 간의 논리적 연결 유지, 연결 접속, 동기제어를 담당하는 OSI 7계층 (5계층)

전송 계층(Transport Layer)
송수신 프로세스 간의 신뢰성 있는 통신을 보장하며 데이터 분할, 재조립, 흐름 제어, 오류 제어, 혼잡 제어를 담당하는 OSI 7계층 (4계층)

네트워크 계층(Network Layer)
단말기 간 데이터를 전송하기 위한 최적화된 경로(라우팅)를 제공하는 OSI 7계층 (3계층)

데이터링크 계층(DataLink Layer)
인접 시스템 간 데이터 전송, 동기화, 오류 제어, 흐름 제어, 회선 제어를 담당하는 OSI 7계층 (2계층)

물리 계층(Physical Layer)
0과 1의 비트 정보를 회선에 보내기 위해 전기적 신호로 변환하는 OSI 7계층 (1계층)

OSI 7계층
국제 표준화 기구인 ISO에서 개발한 컴퓨터 네트워크 프로토콜 디자인과 통신을 계층별로 나누어 설명한 개방형 시스템 상호 연결 모델

처리량(Throughput)
애플리케이션이 주어진 시간에 처리할 수 있는 트랜잭션의 수

응답시간(Response Time)
사용자 입력이 끝난 후, 애플리케이션의 응답 출력이 개시될 때까지의 시간

경과 시간(Turnaround Time)
사용자가 요구를 입력한 시점부터 트랜잭션을 처리 후 그 결과의 출력이 완전히 완료될 때까지 걸리는 시간

자원 사용률(Resource Usage)
애플리케이션이 트랜잭션을 처리하는 동안 사용하는 CPU, 메모리, 네트워크 등의 사용량

애플리케이션 성능 측정 지표
사용자가 요구한 애플리케이션의 기능을 최소한의 자원을 사용해 얼마나 빠르고 많이 처리하는지를 나타내는 기준

BIA(Business Impact Analysis)
장애나 재해로 인한 운영상의 주요 손실을 가정하여 시간 흐름에 따른 영향도 및 손실평가를 조사하는 비즈니스 영향 분석

RTO(Recovery Time Objective)
업무중단 시점부터 업무가 복구되어 다시 가동될 때까지의 목표 복구 시간

RPO(Recovery Point Objective)
업무중단 시점부터 데이터가 복구되어 정상 가동될 때 허용되는 데이터 손실 시점(목표 복구 시점)

DRP(Disaster Recovery Plan)
재난으로 장기간 시설 운영이 불가능한 경우를 대비한 재난 복구 계획

DRS(Disaster Recovery System)
재해복구계획의 원활한 수행을 지원하기 위해 평상시에 인적·물적 자원 및 관리체계를 통합해 확보해 두는 재해복구센터

비즈니스 연속성 계획(BCP)
재난 발생 시 핵심 업무 기능 수행의 연속성을 유지하여 고객 서비스 지속성을 보장하고 신뢰도를 높이는 방법론

AJAX(Asynchronous JavaScript and XML)
브라우저의 XMLHttpRequest 객체를 이용해 전체 페이지를 새로 고치지 않고 페이지 일부의 데이터만 비동기적으로 로드하는 기법

구조적 방법론(Structured Development)
전체 시스템을 기능에 따라 나누어 개발하고 이를 통합하는 분할과 정복 접근 방식의 소프트웨어 개발 방법론

정보공학 방법론(Information Engineering Development)
정보시스템 개발에 필요한 관리 절차와 작업 기반을 체계화한 소프트웨어 개발 방법론

객체지향 방법론(Object-Oriented Development)
'객체'라는 기본 단위로 시스템을 분석 및 설계하는 소프트웨어 개발 방법론

컴포넌트 기반 방법론(CBD : Component Based Development)
소프트웨어를 구성하는 컴포넌트를 조립해서 하나의 새로운 응용 프로그램을 작성하는 소프트웨어 개발 방법론

애자일 방법론(Agile Development)
절차보다는 사람이 중심이 되어 변화에 유연하고 신속하게 적응하면서 시스템을 개발할 수 있는 경량 개발 방법론

제품 계열 방법론(Product Line Development)
특정 제품에 적용하고 싶은 공통된 기능을 미리 정의하여 개발하는 소프트웨어 개발 방법론

인증(AH) 프로토콜
메세지 인증 코드(MAC)를 이용하여 인증과 송신처 인증을 제공하지만 기밀성(암호화)은 제공하지 않는 IPSec 프로토콜

암호화(ESP) 프로토콜
메세지 인증 코드(MAC)와 암호화를 모두 이용하여 인증, 송신처 인증, 그리고 기밀성까지 제공하는 IPSec 프로토콜

키 관리(IKE) 프로토콜
공개된 네트워크를 통하여 암호화 Key를 주고받는 알고리즘과 관리 방법을 정의하는 IPSec 프로토콜

IPSec
IP 계층(3계층)에서 무결성과 인증을 보장하는 AH와 기밀성을 보장하는 ESP를 이용한 IP 보안 프로토콜

Mediator (중재자)
객체 간 통신이 복잡해져 결합도가 높아지는 것을 막기 위해, 통신을 통제하고 지시하는 중재자를 두어 객체 지향의 목표를 달성하는 행위 패턴

Interpreter (해석기)
언어의 구문을 분리하고 해당 구문의 해석을 맡는 클래스를 작성하여 다양한 형태의 언어 구문을 해석할 수 있게 만드는 행위 패턴

Iterator (반복자)
컬렉션 구현 방법을 노출시키지 않으면서도 집합체 안에 있는 모든 항목에 순차적으로 접근할 방법을 제공하는 행위 패턴

Template Method (템플릿 메서드)
전체 일을 수행하는 구조는 바꾸지 않으면서 특정 작업 단계의 처리만 서브 클래스로 캡슐화하여 변경할 수 있게 하는 행위 패턴

Observer (옵서버)
한 객체의 상태가 바뀌면 그 객체에 의존하는 다른 객체들에 연락이 가고 자동으로 내용이 갱신되는 행위 패턴

State (상태)
객체의 상태를 클래스화하여 캡슐화하고 상태에 따라 다르게 행동하도록 그것을 참조하게 하는 행위 패턴

Visitor (방문자)
데이터 구조로부터 처리 기능을 분리하여, 별도의 클래스가 각 데이터 구조를 돌아다니며 특정 작업을 수행하도록 만드는 행위 패턴

Command (명령)
실행될 기능을 캡슐화함으로써 주어진 여러 기능을 재사용성이 높은 클래스로 설계하여 실행할 수 있게 하는 행위 패턴

Strategy (전략)
알고리즘 군을 추상 클래스로 정의하고 각각을 캡슐화하여 필요할 때 서로 교환해서 사용할 수 있게 하는 행위 패턴

Memento (메멘토)
클래스 설계 관점에서 객체의 상태 정보를 저장하고 필요 시 이전 상태로 복구할 수 있도록 적용하는 행위 패턴

Chain of Responsibility (책임 연쇄)
기능 처리에 대한 연결이 정적으로 하드 코딩되는 것을 막고, 동적으로 연결된 객체들을 따라 요청이 다르게 처리될 수 있도록 하는 행위 패턴

디자인 패턴
소프트웨어 설계 시 자주 발생하는 문제에 대해 재사용할 수 있도록 만들어진 훌륭한 해결책 (생성, 구조, 행위 패턴으로 나뉨)

리눅스 기반
안드로이드가 리눅스 커널 위에서 동작한다는 특징

자바와 코틀린 언어
안드로이드에서 고수준 언어를 사용해 응용 프로그램을 작성한다는 특징

런타임 라이브러리
안드로이드가 컴파일된 바이트 코드를 구동하는 기능을 갖췄다는 특징

안드로이드 소프트웨어 개발
응용 프로그램을 개발하는 데 필요한 각종 도구와 API를 제공한다는 특징

안드로이드(Android)
구글에서 개발한 운영체제로 리눅스 위에서 구동하며 휴대용 장치를 위한 미들웨어, 사용자 인터페이스, 표준 응용 프로그램을 포함하는 모바일 운영체제

SOAP(Simple Object Access Protocol)
HTTP, HTTPS, SMTP 등을 사용하여 XML 기반의 메세지를 네트워크 상태에서 교환하는 프로토콜

XSS (Cross Site Script)
검증되지 않은 외부 입력 데이터가 포함된 웹 페이지를 열람할 때, 웹 페이지에 포함된 부적절한 스크립트가 실행되게 하는 보안 취약점 공격

사이트 간 요청 위조 (CSRF)
사용자가 자신의 의지와 무관하게 공격자가 의도한 행위를 특정 웹 사이트에 강제로 요청하게 만드는 공격

SQL 삽입 (Injection)
응용 프로그램의 보안 취약점을 이용해 악의적인 SQL 구문을 삽입하여 DB 접근을 통해 정보를 탈취하거나 조작하는 공격 기법

직관성(Intuitiveness)
누구나 쉽게 이해하고 쉽게 사용할 수 있어야 한다는 UI 설계 원칙

유효성(Efficiency)
정확하고 완벽하게 사용자의 목표가 달성될 수 있도록 제작해야 한다는 UI 설계 원칙

학습성(Learnability)
초보와 숙련자 모두가 쉽게 배우고 사용할 수 있게 제작해야 한다는 UI 설계 원칙

유연성(Flexibility)
사용자의 인터랙션을 최대한 포용하고 실수를 방지할 수 있도록 제작해야 한다는 UI 설계 원칙

LOD(Linked Open Data)
웹상에 존재하는 데이터를 개별 URI로 식별하고, 각 URI에 링크 정보를 부여함으로써 상호 연결된 웹을 지향하는 데이터

개념적 데이터 모델(Conceptual Data Modeling)
현실 세계에 대한 인식을 추상적, 개념적으로 표현하여 개념적 구조를 도출하는 데이터 모델링의 첫 단계

논리적 데이터 모델(Logical Data Modeling)
업무의 모습을 모델링 표기법으로 형상화하여 사람이 이해하기 쉽게 표현한 데이터 모델링 단계

물리적 데이터 모델(Physical Data Modeling)
논리 데이터 모델을 특정 DBMS의 특성 및 성능을 고려하여 실제 물리적인 스키마를 만드는 데이터 모델링의 마지막 단계

데이터 모델링의 구성요소
현실 세계를 데이터베이스에 표현하기 위한 핵심 요소 3가지인 개체(Entity), 속성(Attribute), 관계(Relation)
