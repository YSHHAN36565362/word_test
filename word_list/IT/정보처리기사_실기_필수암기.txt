추상 팩토리 (Abstract Factory) - 생성 패턴 (5개)
치트키 단어: 연관된 객체들의 그룹/패밀리/조합

빌더 (Builder) - 생성 패턴 (5개)
치트키 단어: 생성과 표현의 분리, 단계별 생성

팩토리 메서드 (Factory Method) - 생성 패턴 (5개)
치트키 단어: 서브 클래스에서 객체 생성, 생성 위임

프로토타입 (Prototype) - 생성 패턴 (5개)
치트키 단어: 원본 객체 복제/복사

싱글톤 (Singleton) - 생성 패턴 (5개)
치트키 단어: 오직 하나의 인스턴스/객체

어댑터 (Adapter) - 구조 패턴 (7개)
치트키 단어: 호환성 없는 인터페이스 변환, 변환기

브릿지 (Bridge) - 구조 패턴 (7개)
치트키 단어: 추상화와 구현의 분리, 독립적 확장

컴포지트 (Composite) - 구조 패턴 (7개)
치트키 단어: 단일 객체와 복합 객체의 동일시, 트리 구조

데코레이터 (Decorator) - 구조 패턴 (7개)
치트키 단어: 동적인 기능 추가/장식

퍼사드 (Facade) - 구조 패턴 (7개)
치트키 단어: 통합 인터페이스 제공, 복잡한 시스템의 단순화

플라이웨이트 (Flyweight) - 구조 패턴 (7개)
치트키 단어: 객체의 공유, 메모리 절약

프록시 (Proxy) - 구조 패턴 (7개)
치트키 단어: 대리인/대리 객체, 접근 제어

책임 연쇄 (Chain of Responsibility) - 행위 패턴 (11개)
치트키 단어: 동적으로 연결, 요청을 다음 객체로 전달

커맨드 (Command) - 행위 패턴 (11개)
치트키 단어: 실행될 기능을 캡슐화, 명령의 캡슐화

인터프리터 (Interpreter) - 행위 패턴 (11개)
치트키 단어: 언어의 구문 해석, 문법 표현

이터레이터 (Iterator) - 행위 패턴 (11개)
치트키 단어: 구현 방법을 노출하지 않고 순차적 접근

중재자 (Mediator) - 행위 패턴 (11개)
치트키 단어: 중재자를 둠, 통신의 빈도를 줄임, 느슨한 결합

메멘토 (Memento) - 행위 패턴 (11개)
치트키 단어: 객체의 정보를 저장, 이전 상태 복구

옵저버 (Observer) - 행위 패턴 (11개)
치트키 단어: 상태가 바뀌면 자동으로 연락/갱신, 일대다 의존성

상태 (State) - 행위 패턴 (11개)
치트키 단어: 객체 상태를 캡슐화, 상태에 따른 행동 변경

전략 (Strategy) - 행위 패턴 (11개)
치트키 단어: 알고리즘 군을 정의, 교환해서 사용

템플릿 메서드 (Template Method) - 행위 패턴 (11개)
치트키 단어: 전체 구조는 바꾸지 않고, 서브 클래스로 캡슐화

방문자 (Visitor) - 행위 패턴 (11개)
치트키 단어: 데이터 구조로부터 처리 기능을 분리, 클래스를 돌아다님

내용 결합도
치트키 단어: 직접, 내부 (한 모듈이 다른 모듈의 내부 기능 및 그 내부 자료를 직접 참조하거나 수정)

공통 결합도
치트키 단어: 전역 변수, 공통 데이터 (여러 모듈이 공통 데이터 영역을 정해서 공동으로 참조)

외부 결합도
치트키 단어: 외부, 프로토콜/디바이스 (외부에서 도입된 데이터 포맷, 통신 프로토콜, 디바이스 인터페이스를 공유)

제어 결합도
치트키 단어: 어떻게 처리를 해야 한다는 요소, 제어 (단순 처리할 대상인 값뿐만 아니라 어떻게 처리해야 한다는 제어 요소가 전달됨)

스탬프 결합도
치트키 단어: 배열, 객체, 구조 (모듈 간의 인터페이스로 배열이나 객체, 구조 등 복합 데이터가 전달됨)

자료 결합도
치트키 단어: 파라미터(인자/단순 값)만 (모듈 간의 인터페이스로 전달되는 파라미터나 인자를 통해서만 상호작용)

제시해주신 최신 보안 및 신기술 용어 정보를 요청하신 포맷(단어 제목 + 치트키 단어)에 맞춰 깔끔하게 정리했습니다.

랜섬웨어 (Ransomware)
치트키 단어: 파일 암호화, 금전(몸값) 요구

살충제 패러독스 (Pesticide Paradox)
치트키 단어: 동일한 테스트 케이스, 반복적 테스트, 새로운 버그 찾지 못함

스미싱 (Smishing)
치트키 단어: 문자메시지(SMS), 악성 앱 설치 유도

파밍 (Pharming)
치트키 단어: 정상 주소 입력, 가짜 사이트로 위조 접속

랜드 어택 (Land Attack)
치트키 단어: 출발지 IP와 목적지 IP를 같게

스니핑 (Sniffing)
치트키 단어: 몰래 들여다보는, 훔쳐보는, 수동적 공격

스푸핑 (Spoofing)
치트키 단어: 패킷 헤더 위조, MAC 주소 위조

데이터 마이닝 (Data Mining)
치트키 단어: 대규모 데이터, 통계적 규칙이나 패턴 추출, 광석 추출 비유

블록체인 (Blockchain)
치트키 단어: 분산 컴퓨팅, 데이터 위변조 방지, P2P 방식

JSON (JavaScript Object Notation)
치트키 단어: 비동기 통신(AJAX), 속성-값 쌍, 키-값 쌍

XML (Extensible Markup Language)
치트키 단어: 시스템 간 데이터 연계, 데이터 구조를 동일한 형태로 정의

하둡 (Hadoop)
치트키 단어: 분산 파일 시스템(HDFS), 맵리듀스(MapReduce), 대용량 데이터 병렬 처리

스머프 공격 (Smurf Attack)
IPv4와 ICMP의 특성을 악용하여, 송신 주소를 공격 대상의 IP 주소로 위조(스푸핑)한 패킷을 라우터의 브로드캐스트 주소로 전송하여 엄청난 양의 반사 데이터를 집중적으로 보내 네트워크를 마비시키는 공격 기법이다. (키워드: ICMP, 브로드캐스트 주소, 위조된 송신 주소로 대량 응답 반사)

싱글 플러딩 (SYN Flooding)
TCP 연결 과정에서 3-Way Handshake의 취약점을 악용하여, 공격자가 수많은 SYN 패킷만 보내고 서버가 보낸 SYN+ACK 패킷에는 응답하지 않아 서버의 접속 대기 큐(Queue)를 가득 채워 마비시키는 공격 기법이다. (키워드: TCP, SYN 패킷, Handshake 취약점, 대기 큐 마비)

죽음의 핑 (Ping of Death)
인터넷 규정 크기보다 훨씬 큰 대형 ICMP 패킷을 의도적으로 분할하여 전송함으로써, 수신 측에서 이 패킷들을 다시 조립하는 과정에서 과부하가 걸리거나 버퍼 오버플로우가 발생하여 시스템을 다운시키는 공격 기법이다. (키워드: 허용 범위 이상의 큰 패킷, ICMP 패킷 분할, 재조립 과부하)

티어드롭 (Teardrop)
IP 패킷을 분송할 때 오프셋(Offset) 값을 교묘하게 조작하여, 수신 측에서 분할된 패킷들을 다시 조립할 때 조립 위치가 겹치거나 오류가 나도록 만들어 시스템을 다운시키는 공격 기법이다. (키워드: IP 패킷 오프셋 조작, 겹치거나 어긋난 재조립 오류)

루디 (RUDY / R-U-Dead-Yet)
HTTP POST 요청 시 헤더의 Content-Length(데이터 크기)를 엄청나게 크게 설정해 두고, 실제 본문 데이터는 아주 소량(1바이트씩)으로 매우 천천히 전송하여 서버가 연결을 끊지 못하고 계속 대기하게 만들어 자원을 고갈시키는 공격 기법이다. (키워드: HTTP POST, Content-Length 크게 설정, 아주 천천히 데이터 전송)

슬로로리스 (Slowloris)
HTTP 요청 헤더의 끝을 알리는 개행 문자(\r\n\r\n)를 온전하게 보내지 않고 완전히 끝내지 않은 채로 계속 연결을 유지하여, 서버의 웹 서버 커넥션 자원을 모두 차지해 정상적인 서비스를 불가능하게 만드는 공격 기법이다. (키워드: HTTP 요청 헤더 끝을 알리지 않음, 불완전한 헤더로 연결 유지)

디도스 (DDoS / 분산 서비스 거부 공격)
감염된 수많은 좀비 PC(봇넷)들을 원격으로 조종하여, 특정 타깃 시스템이나 사이트에 한꺼번에 엄청난 양의 트래픽을 집중시킴으로써 서비스를 완전히 마비시키는 공격 기법이다. (키워드: 여러 대의 좀비 PC/봇넷, 대량의 트래픽 집중 분산 공격)

네트워크 핵심 개념인 패킷 교환 방식과 라우팅 프로토콜을 요청하신 서식에 맞춰 깔끔하게 정리해 드릴게요. 시험 직전에 눈으로 슥 읽으면서 복습하기 좋은 치트키 단어들 위주로 구성했습니다.

가상회선 (Virtual Circuit)
데이터를 보내기 전에 출발지부터 목적지까지 가상의 전용 선로를 미리 깔아두어 모든 패킷이 똑같은 길로 순서대로 이동하는 연결형 교환 방식이다. (키워드: 연결형, 미리 경로 설정, 기차 선로 비유)

데이터그램 (Datagram)
미리 길을 예약하지 않고 패킷마다 목적지 주소를 붙여 도로에 던져서 각자 알아서 가장 빠른 길로 찾아가게 하는 비연결형 교환 방식이다. (키워드: 비연결형, 패킷마다 독립적 이동, 택배 트럭 비유)

IGP (Interior Gateway Protocol)
독립된 하나의 자율 시스템(AS) 또는 도메인 내부의 라우터들끼리 경로 정보를 교환할 때 사용하는 내부 라우팅 프로토콜이다. (키워드: 도메인 내부 경로 설정)

EGP (Exterior Gateway Protocol)
서로 다른 도메인이나 자율 시스템(AS) 사이에서 라우팅 정보를 교환할 때 사용하는 외부 라우팅 프로토콜이다. (키워드: 다른 도메인 사이, 보안과 제어가 목적)

OSPF (Open Shortest Path First)
대규모 네트워크에 적합하도록 RIP의 단점을 개선한 라우팅 프로토콜로, 다익스트라 알고리즘을 기반으로 최단 경로를 찾는 링크 상태 프로토콜이다. (키워드: RIP 단점 개선, 복잡하고 규모가 큰 네트워크)

BGP (Border Gateway Protocol)
자율 시스템(AS) 간의 라우팅 정보를 교환하는 외부 게이트웨이 프로토콜로, 라우팅 루프를 제거하고 정책에 따라 최적의 경로를 결정한다. (키워드: AS 연결 그래프, 라우팅 루프 제거, 정책 결정)
